URLs page of settings window

Available URLs section of URLs page shows the list of URL records that can be used to open tasks. It allows to create, modify and remove records from the list.
By default open tasks in section allows to choose which URL record will be used to open task(s) when user uses buttons on main window and systray menu commands.

For each URL it is possible to define:

  • URL caption
  • Link to open task
  • Link to open child tasks
  • Link to open related tasks
  • Login, Password
  • 4 hotkeys to be used to open tasks, child, parent and related tasks correspondingly
  • Where SoftTest defects (that are considered as special type of tasks) should be opened: on ISD QA site or in SoftTest application

NOTE: When you use a hotkey, a task name is read from Windows Clipboard, not from "Task Name" field.

There is a useful feature that allows to pass login and password automatically to the browser when you open a site, if required.
To use this feature you can enter your login and password to corresponding fields listed above.
You can also enter your login and password to URL directly in the following form:

http://login:password@url

If login and password are entered in URL in the format above they will be used instead of login and password entered in "Login" and "Password" fields.

Please note that:

  • Recent versions of Microsoft Internet Explorer have this feature DISABLED by default (see http://support.microsoft.com/kb/834489). TMS Launcher will activate the feature automatically asking for confirmation from user.
  • Microsoft Internet Explorer 9.0 has this feature DISABLED and TMS Launcher cannot enable it.
  • Mozilla Firefox asks for confirmation from user every time it opens URL in format above.
    To disable this confirmation you can do the following:
    • Type "about:config" in the location bar and press Enter
    • Right-click in the empty space and select New|Integer
    • Input "network.http.phishy-userpass-length" as the name and "1024" as the value
