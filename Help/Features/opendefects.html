Opening SoftTest defects

SoftTest defects are considered by TMS Launcher as special type of TMS tasks. TMS tasks with client names that match with predefined values are supposed to be defects.
Main window buttons and hotkeys, specified on URLs page of settings window for all available URLs, can be used to open defects, child and parent defects.

Defects can be opened in two applications:

  • In a browser on ISD QA site
    In this case, for each predefined value user can assign a SoftTest project from which defect will be read and an URL to open defect. Described settings can be made on Defects page of settings window.
  • In SoftTest application directly
    In this case, settings from Defects page are used as well. However, since defects are opened using main window buttons and hotkeys, specified for all URLs, it is possible to specify where defects can be opened for each such URL.
    For example, you can specify that defects will be opened on ISD QA site when using "Usual TMS" URL and in SoftTest when using "Alternative TMS" URL.
    NOTE: if you open several defects from the same project simultaneuosly, only one SoftTest application will be started and the defects will be opened in it.

Also there is SoftTest page in settings window that allows to specify some settings related to SoftTest.
